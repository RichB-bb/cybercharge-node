# Navigation And Returns Report

## Navbar Changes

- Reduced desktop navigation to four primary destinations:
  - Infrastructure
  - Network
  - Payment
  - Dashboard
- Removed outdated links to sections that are no longer part of the simplified homepage flow.
- Added a primary `Purchase Allocation` CTA on the right side of the desktop navigation.
- `Infrastructure` now scrolls to the Real Charging Infrastructure section.
- `Network` scrolls to the Deployment Map section.
- `Payment` and `Purchase Allocation` scroll to the Payment section.
- `Dashboard` links directly to `/dashboard`.

## Mobile Improvements

- Mobile menu now mirrors the simplified four-item structure.
- Removed low-priority menu items to reduce decision friction.
- Added a prominent `Purchase Allocation` CTA inside the mobile menu.
- The menu closes after selecting a navigation item or CTA.

## Returns Explanation Design Logic

- Added a compact `How Returns Are Generated` section after Payment.
- Kept the explanation intentionally short and compliance-safe.
- Used three simple cards:
  - Infrastructure Allocation
  - Charging Activity
  - Dashboard Tracking
- Avoided APY, ROI, guaranteed return, fixed return, profit percentage, and yield language.

## Conversion Lift Points

- Users now see a navigation model that matches the actual homepage structure.
- The purchase CTA is visible without forcing users to discover the Payment link.
- The returns explanation is placed after the payment entry, so it supports confidence without delaying conversion.
- Mobile users face fewer choices and a clearer path toward payment.

## Expected User Path Change

Before:

Hero -> multiple explanatory sections -> map -> long payment flow -> payment

After:

Hero -> infrastructure proof -> asset narrative -> deployment map -> payment entry -> short returns explanation -> footer

The path is shorter, clearer, and better aligned with a Tesla / Apple style product flow.
